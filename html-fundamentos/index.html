HTML Fundamentos

Estructura base de una página web

Toda página web tiene la misma estructura:

<header>  → cabecera (logo, navegación)
<main>    → contenido principal
<footer>  → pie de página

Header y navegación

El header siempre contiene un <nav> con la navegación. Los links de navegación siempre van en listas:

<header>
  <nav>
    <a class="logo" href="/">Angel García</a>
    <ul>
      <li><a href="#sobre-mi">Sobre mí</a></li>
      <li><a href="#contacto">Contacto</a></li>
    </ul>
  </nav>
</header>
  • <nav> → navegación. Puede haber más de uno en la página (header, footer).
  • <ul> + <li> → lista de items de navegación. Se usa aunque visualmente no parezca una lista.
  • href="/" → va al inicio del sitio.
  • href="#sobre-mi" → hace scroll a un elemento con id="sobre-mi" en la misma página.
  • href="/proyectos" → va a otra página.

Si hay submenús, se anidan listas dentro de <li>:

<li>
  <a href="#proyectos">Proyectos</a>
  <ul>
    <li><a href="#sql">SQL</a></li>
    <li><a href="#python">Python</a></li>
  </ul>
</li>

Semántica

Las etiquetas semánticas le dicen al navegador qué es cada parte:

Etiqueta Significado
<header> Cabecera de la página
<nav> Navegación
<main> Contenido principal
<section> Bloque temático con contenido propio
<footer> Pie de página

Importan para: lectores de pantalla (accesibilidad), Google (SEO) y legibilidad del código.

Secciones

<main> contiene <section>. Cada sección representa un tema distinto. No hay límite de secciones.

Regla práctica: si le pondrías un título diferente, es una sección diferente.

<main>
  <section id="hero">...</section>
  <section id="sobre-mi">...</section>
  <section id="proyectos">...</section>
  <section id="contacto">...</section>
</main>

Jerarquía de títulos

Solo puede haber un <h1> por página. Las secciones usan <h2>, los subtemas <h3>, etc.

h1 → título de la página      (solo uno)
h2 → títulos de secciones     (varios)
h3 → subtítulos dentro de h2
...hasta h6

Aplica igual en Markdown: un solo # por documento, secciones con ##.

Contenedores: div y span

Ambos no tienen significado semántico — existen solo para el CSS.

Etiqueta Tipo Cuándo usarla
<div> Bloque (ocupa toda la línea) Agrupar bloques de contenido para layout
<span> Inline (vive dentro del texto) Estilizar un trozo de texto dentro de un párrafo 
<!-- div: para columnas, grupos, layout -->
<div class="columnas">
  <div class="izquierda">...</div>
  <div class="derecha">...</div>
</div>

<!-- span: para texto inline -->
<p>El resultado fue <span class="positivo">+342</span> este mes.</p>

Links: <a>

<a href="/">Inicio</a>                              <!-- va al inicio -->
<a href="#contacto">Contacto</a>                    <!-- scroll en la misma página -->
<a href="/proyectos">Proyectos</a>                  <!-- otra página -->
<a href="mailto:email@gmail.com">Correo</a>         <!-- abre cliente de correo -->
<a href="https://youtube.com" target="_blank">YT</a> <!-- abre en nueva pestaña -->

Un <a> puede verse como botón aplicándole una clase CSS — sigue siendo un link, no un <button>.

Regla: <a> cuando lleva a algún lugar. <button> cuando ejecuta una acción (formulario, JavaScript).

HTML y frameworks

HTML siempre está presente. Frameworks como React o Vue escriben el código de otra forma, pero el navegador siempre recibe HTML puro al final.

Los frameworks resuelven dos problemas que HTML puro no maneja bien: - Repetición → no escribir el mismo bloque cientos de veces (listas dinámicas de videos, proyectos, etc.) - Estado → HTML que reacciona a lo que hace el usuario sin recargar la página

Para webs estáticas y simples, HTML puro es suficiente y más directo.
