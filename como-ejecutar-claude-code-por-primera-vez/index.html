Cómo ejecutar Claude Code por primera vez

datablog claude-code
Actualizado: 2026-06-09

1. ¿Desde dónde ejecutar claude?

Cuando ejecutas claude, Claude Code tiene acceso al directorio desde el que lo lanzaste y todo lo que hay dentro. Eso define cuánto puede ver y tocar sin que se lo pidas explícitamente.

Si lo ejecutas desde tu directorio home (~): Claude tiene acceso implícito a todo tu usuario — documentos, descargas, proyectos, configuraciones, todo. Puede leer, editar y ejecutar cualquier archivo sin que se lo indiques en medio de una sesión si lo cree conveniente.

Si lo ejecutas desde una carpeta específica: Solo puede navegar dentro de esa carpeta. Si necesita buscar en otro lugar, tienes que decírselo explícitamente.

# ❌ Desde el home — acceso implícito a todo tu usuario
cd ~
claude

# ✅ Desde una carpeta específica — acceso limitado a esa carpeta
cd ~/proyectos/mi-proyecto
claude

Recomendación personal: siempre ejecuta Claude Code desde la misma carpeta fija, independientemente del proyecto en el que estés trabajando. Aunque tu proyecto esté en la carpeta F, lanza Claude siempre desde la carpeta A. Los beneficios de hacer esto — relacionados con tener a Claude como un agente con memoria entre sesiones — los cubrimos más adelante en esta serie.

2. Primera vez que ejecutas claude

Claude Code usa tu cuenta de Anthropic para autenticarse. Necesitas mínimo el plan Pro — el plan gratuito no incluye acceso a Claude Code. No tienes que configurar nada manualmente — el proceso es automático vía navegador.

# Navega a tu carpeta de lanzamiento
cd ~/tu-carpeta

# Ejecuta claude
claude

Lo que ocurre en orden:

  1. Claude Code detecta que no hay sesión activa
  2. Se abre tu navegador automáticamente
  3. Inicias sesión con tu cuenta de Anthropic
  4. Autorizas el acceso
  5. El navegador te confirma que puedes cerrar la pestaña
  6. Tu terminal muestra la confirmación de seguridad del workspace:
Accessing workspace:
/Users/tu_usuario/tu-carpeta

Quick safety check: Is this a project you created or one you trust?
Claude Code'll be able to read, edit, and execute files here.

❯ 1. Yes, I trust this folder
  2. No, exit

Confirmas con 1 y Claude Code queda listo para usar.

⚠️ La autenticación con Anthropic solo ocurre una vez. Queda guardada en ~/.claude y no te la pedirá de nuevo aunque cierres la terminal o reinicies la Mac.

3. Las siguientes veces que ejecutas claude

La autenticación ya está guardada. Lo único que ocurre es la confirmación del workspace si es un directorio nuevo:

En un directorio ya confiado → entra directo:

cd ~/tu-carpeta
claude
# Entra directamente sin preguntas

En un directorio nuevo → te pregunta si confías:

cd ~/nuevo-directorio
claude
# Muestra la confirmación de seguridad del workspace

La confirmación de seguridad aparece una vez por directorio. Una vez que confirmas, Claude Code recuerda ese directorio.

⚠️ Para Claude Code, proyecto = ruta exacta. Siempre entra desde la raíz de tu carpeta de lanzamiento, no desde un subdirectorio. De lo contrario se crearán sesiones separadas aunque sea el mismo proyecto.

Salir de Claude Code

# Opción 1 — comando
/exit

# Opción 2 — atajo de teclado
Ctrl + C

5. Conceptos clave antes de continuar

5.1 Sesión

Una sesión es una conversación con Claude en un directorio específico. Cada vez que ejecutas claude en una carpeta, estás iniciando o retomando una sesión de ese directorio. Se guarda automáticamente en ~/.claude/projects/ — no dentro de tu proyecto real.

~/.claude/projects/
└── Users-angel-mi-proyecto/     ← carpeta que representa tu proyecto
    └── *.jsonl                  ← archivos de sesión en formato JSON Lines

Lo que guarda cada .jsonl no es una transcripción literal — es un registro estructurado que incluye los mensajes, las modificaciones de archivos realizadas y metadatos como timestamp y directorio. Es lo que Claude Code usa para retomar una sesión anterior con /resume.

Las sesiones se guardan automáticamente. No hay una acción explícita de "guardar" — ocurre sin que lo decidas.

⚠️ Las sesiones crecen con el tiempo. Revisa periódicamente cuánto ocupa ~/.claude/projects/ — puede ser considerable con el uso.

5.2 Archivos que se cargan al iniciar una sesión

Cuando ejecutas claude, antes de que puedas escribir nada, Claude ya cargó automáticamente un conjunto de archivos de contexto. Los principales son:

  • CLAUDE.md — instrucciones persistentes para Claude (global y de proyecto)
  • settings.json — configuración de permisos y comportamiento (global y de proyecto)
  • Archivos de auto memory — notas acumuladas de sesiones anteriores

Es posible que existan otros archivos que se carguen al inicio de sesión. Lo que está confirmado son estos tres. Cada uno tiene su propio post en esta serie.

5.3 CLAUDE.md

Es el archivo más importante de los tres. Le da a Claude instrucciones persistentes que aplican en cada sesión — sin necesidad de repetirlas cada vez. Existe en dos niveles:

Nivel usuario (~/.claude/CLAUDE.md): Instrucciones globales. Se cargan en cualquier proyecto que abras con Claude Code.

Nivel proyecto (~/mi-proyecto/.claude/CLAUDE.md): Instrucciones específicas de ese proyecto. Tienen precedencia sobre las globales cuando hay conflicto.

5.4 Auto memory

Notas que Claude escribe solo sobre tus preferencias y patrones de trabajo a medida que interactúas con él. Se guardan en ~/.claude/projects/<proyecto>/memory/ y se cargan automáticamente al iniciar una sesión en ese proyecto.

A diferencia de CLAUDE.md — que tú escribes — el auto memory lo genera Claude sin que se lo pidas.

6. Verificar que estás autenticado

claude doctor

Busca esta línea en el output:

✓ Logged in as: tu@email.com

7. Cerrar sesión

claude logout

Útil si necesitas cambiar de cuenta o si alguien más va a usar Claude Code en el mismo usuario de Mac.

Gotchas

  • Ejecutar claude desde el home directory le da acceso implícito a todo tu usuario. Siempre usar una carpeta fija de lanzamiento.
  • Proyecto = ruta exacta. Entrar desde un subdirectorio crea una sesión separada aunque sea el mismo proyecto. Siempre desde la raíz.
  • Las sesiones crecen con el uso. Revisar periódicamente el tamaño de ~/.claude/projects/ — puede ocupar espacio considerable con el tiempo.
